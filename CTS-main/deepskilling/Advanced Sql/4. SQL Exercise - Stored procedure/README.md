# Employee Management System - Stored Procedures

## Objective

The objective of this experiment is to understand and implement Stored Procedures in Microsoft SQL Server. The exercises demonstrate creating, modifying, executing, deleting, and enhancing stored procedures using parameters, transactions, dynamic SQL, output parameters, and error handling.

---

## Software Used

- Microsoft SQL Server
- SQL Server Management Studio (SSMS)

---

## Database Tables

- Departments
- Employees

---

## Exercises Performed

### Exercise 1
Created a stored procedure to retrieve employee details based on Department ID.

### Exercise 2
Modified the stored procedure to include employee salary.

### Exercise 3
Deleted an existing stored procedure.

### Exercise 4
Executed a stored procedure with parameters.

### Exercise 5
Created a stored procedure to count employees in a department.

### Exercise 6
Used an OUTPUT parameter to return the total salary of a department.

### Exercise 7
Created a stored procedure to update employee salary.

### Exercise 8
Implemented conditional logic to give bonuses to employees.

### Exercise 9
Used transactions to ensure data integrity while updating salaries.

### Exercise 10
Implemented Dynamic SQL to search employee records using flexible filters.

### Exercise 11
Handled runtime errors using TRY...CATCH blocks and displayed custom error messages.

## Advantages of Stored Procedures

- Improves code reusability.
- Enhances database security.
- Reduces network traffic.
- Improves performance.
- Simplifies maintenance.
- Supports transactions and error handling.
